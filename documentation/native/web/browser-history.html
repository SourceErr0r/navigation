Setup Browser History

Browser History

On native, itâ€™s up to you what happens when the user presses the back button. But on the web, the browser back button always steps back to the previously visited URL. Because browser history is linear, you can only connect it up to one NavigationStack. In an app that has multiple stacks, like the 'inbox' and 'contacts' stacks in our example email app, the quickest way to get up and running is to disable browser history. That means stopping and disabling the history managers on the associated state navigators before making your first navigate call. 

inboxNavigator.historyManager.disabled = true;
inboxNavigator.historyManager.stop();

contactsNavigator.historyManager.disabled = true;
contactsNavigator.historyManager.stop();

Enabling the Browser Back Button

<NavigationBar
  onNavigationPress={() => {
    if (Platform.OS !== 'web') stateNavigator.navigateBack(1);
    else history.back();
}} />
